## Amplicode 2026.1.2

В этом релизе основной фокус был на Spring Skills: мы расширили набор навыков для разработки Spring-приложений и сделали их проще доступными для пользователей разных AI-агентов.

Главное обновление — новый [Spring Data JDBC skill](https://github.com/Amplicode/spring-skills/tree/main/skills/spring-data-jdbc). Он помогает работать с проектами на Spring Data JDBC: учитывать особенности маппинга, репозиториев и доменной модели, а также генерировать код в стиле, совместимом с этим стеком.

Все [Spring Skills](https://github.com/Amplicode/spring-skills) теперь опубликованы в популярных каталогах и реестрах навыков для AI-агентов:

* [skills.sh](https://skills.sh/)
* [agentskill.sh](https://agentskill.sh/)
* [tessl.io/registry](https://tessl.io/registry)
* [lobehub.com](https://lobehub.com)

Мы также значительно улучшили производительность Amplicode на больших проектах. Работа с крупными Spring-кодовыми базами стала быстрее и стабильнее: меньше задержек при анализе проекта, навигации и выполнении операций, которые требуют глубокого понимания структуры приложения.

Кроме того, в релиз вошло множество исправлений ошибок и небольших улучшений, повышающих стабильность повседневной работы.

## Amplicode 2026.1.0

В этот релиз вошли [Spring Agent Toolkit](https://amplicode.ru/ai-spring-agent/), обновлённый вывод в HTTP-клиенте и AOT-метаданные для Spring Data репозиториев.

### Spring Agent Toolkit

Amplicode даёт агенту набор специализированных инструментов для работы со Spring Boot проектами: код генерируется быстрее, соответствует best practices и обходится дешевле — если вы платите за токены.

[![](https://i3.ytimg.com/vi/LUyVgOhMsXI/maxresdefault.jpg)](http://www.youtube.com/watch?v=LUyVgOhMsXI)

**MCP-инструменты** предоставляют агенту структурированный доступ к проекту: список бинов, поиск эндпоинтов, структура JPA-сущностей, конфигурация Security, Docker Compose, Kafka, тесты. Вместо чтения файлов вслепую агент получает точечные запросы — меньше токенов, точнее результат.

[**Spring SKILLS**](https://github.com/Amplicode/spring-skills/tree/main) — готовые инструкции для агента на основе реальных задач разработки: создать entity с миграцией, добавить эндпоинт с учётом Security, сгенерировать DTO и маппер. Подключается к любому совместимому агенту или IDE: Cursor, Windsurf, Claude Code, Codex, JetBrains AI, Copilot, GigaCode, Veai, KodaCode, OpenCode, Kilo Code, Qwen.

### Connekt (HTTP-клиент)

**Компактный вывод в консоли**

Каждый HTTP-запрос и его ответ теперь группируются в единый сворачиваемый блок. По умолчанию блок свёрнут — видна только строка-заголовок: иконка состояния, метод, URL и код ответа. Состояние отображается иконкой по классу HTTP-статуса: 1xx, 2xx, 3xx, 4xx/5xx.

Заголовки запроса и ответа сворачиваются в отдельные под-блоки, тело запроса — отдельно. При выполнении use-case-скрипта с несколькими запросами каждый образует свой элемент.

![](https://storage.yandexcloud.net/amplicode-site/articles/2026-1/folding.png)

### Spring Data: AOT-метаданные для репозиториев

Функциональность работает для Spring Boot 3+ проектов.

**Inlay hints** над методами репозиториев показывают запрос, который выполнится в рантайме:

- Spring Data JPA: JPQL из AOT JSON (если данные доступны), либо вычисляется напрямую при включённой опции `alwaysShowJpaQuery`. Клик открывает меню: извлечь JPQL в аннотацию, настроить, определить Entity Graph.
- Spring Data JDBC: SQL из AOT JSON. Клик предлагает вставить запрос в `@Query`.

Если AOT-данных нет — хинт предлагает собрать проект.

**Gutter-иконки** в левом поле редактора позволяют переходить между интерфейсом репозитория и AOT-сгенерированным классом реализации (`*Impl__AotRepository`) — и обратно. Работает как для классов, так и для отдельных методов.
