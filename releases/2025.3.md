## Amplicode 2025.3.2

В этот релиз вошли новые возможности для Connekt и улучшения в интеграции с DBeaver.

### Совместимость с IntelliJ IDEA 2026.1

Добавлена поддержка IntelliJ IDEA версии 2026.1.

### Connekt (HTTP-клиент)

**Импорт скриптов через `@Import`**

Теперь в Connekt-скриптах можно переиспользовать код из других скриптов с помощью аннотации `@file:Import`. Основной сценарий — вынесение аутентификации в отдельный файл:

```kotlin
// oauth.connekt.kts
val ssoHost: String by env

val token by oauth(
    "$ssoHost/realms/master/protocol/openid-connect/auth",
    "client", null, "openid",
    "$ssoHost/realms/master/protocol/openid-connect/token",
    "http://localhost:28080/callback"
)
```

```kotlin
// hello.connekt.kts
@file:Import("oauth.connekt.kts")

val host = "http://localhost:8080"

GET("$host/hello") {
    bearerAuth(token.accessToken)
} then {
    val bodyValue = body?.string()
    assert(bodyValue == "Hello1")
}
```

Из импортирующего скрипта доступны типы и переменные только непосредственного родителя — транзитивный доступ не поддерживается. Автодополнение имени файла в аннотации `@Import` работает через basic completion.

**Приватное окружение (Private Environment)**

Добавлена поддержка приватного файла окружения — по аналогии с другими HTTP-клиентами. Приватный файл создаётся рядом с публичным и переопределяет его значения (а также добавляет новые ключи). При создании файла Amplicode предложит добавить его в `.gitignore`.

Подробнее про работу с приватными переменными окружения читайте в статье на Хабр: https://habr.com/ru/companies/haulmont/articles/990732/

**Создание Connekt-скриптов из Amplicode Explorer**

Теперь новые Connekt-скрипты можно создавать прямо из дерева Amplicode Explorer, нажатием правой кнопки мыши по секции "Connekt".

## Amplicode 2025.3.1

В этот релиз вошла одна, но очень классная фича **Ctrl+C → Ctrl+V на стероидах**. Теперь при копировании кода Amplicode автоматически добавляет нужную инжекцию бинов. С учётом контекста, `@Primary`, `@Qualifier`, дженериков, `@Bean`-методов, Java и Kotlin — без ручной возни после вставки. Подробнее [тут](https://t.me/amplicode/289).

Спасибо всем, кто кидал баги и стектрейсы в [чат](https://t.me/amplicode_chat)! 

Часть из них уже в релизе:
— генерация OpenAPI схемы больше не падает с исключением
— Шорткат Execute Query At Caret больше не конфликтует с экшенами в IntelliJ IDEA Ultimate
— запуск Connekt-скрипта по шорткату больше не приводит к ProhibitedAnalysisException

Заодно подчистили несколько эксепшенов и багов связанных с [Connekt](https://amplicode.ru/http-client/) и интеграцией с [DBeaver](https://amplicode.ru/db-client/).

## Amplicode 2025.3.0

### Spring MCP от Amplicode (Beta)
- Добавлена beta-поддержка Spring MCP (не включена по умолчанию)
- Возможность интеграции Amplicode с внешними LLM / AI-помощниками
- Инструменты для более корректной и контекстной работы LLM с Spring-проектами

### MyBatis
- Добавлена модель данных MyBatis в Amplicode Explorer
- Реализована навигация по MyBatis mapper'ам
- Поддержан автовайринг MyBatis mapper'ов через автодополнение
- Добавлено автодополнение `mapTo` для доменных объектов
- Реализовано делегирование методов из MyBatis SQL Mapper

### Spring Data JDBC
- Добавлена поддержка фильтров на базе Criteria API
- Улучшен автовайринг бинов в кастомных репозиториях
- Улучшена работа со структурой DTO
- Расширен маппинг для вложенных DTO
- Упрощено создание кастомных Spring Data Repository
- Сокращено количество шаблонного кода при работе с репозиториями

### Connekt (HTTP-клиент)
- Добавлена поддержка комментариев к HTTP-запросам
- Реализованы power asserts с детальной диагностикой ошибок
- Автоматическая генерация paging-параметров (`page`, `size`, `sort`) для pageable-эндпоинтов

### Terraform
- В Amplicode Explorer отображаются поддерживаемые Terraform-ресурсы
- Добавлены генераторы ресурсов Yandex Cloud:
    - Managed Service for Kafka
    - Object Storage
    - Lockbox
    - Certificate Manager
- Добавлены генераторы Kubernetes-ресурсов:
    - Namespace
    - Secret
- Добавлено действие для подключения YC Kubernetes-кластера в локальный kubeconfig
- Поддержан Helm-провайдер и чарты:
    - External Secret Operator
    - Ingress Nginx
    - Envoy Gateway
    - ExternalDNS (Yandex Cloud)
- Добавлена полноценная работа с секретами через External Secrets Operator и Lockbox:
    - SecretStore
    - ClusterSecretStore
    - ExternalSecret для Lockbox

### Helm и Kubernetes
- Поддержан Gateway API при создании Helm-чартов для Spring Boot
- Добавлено автодополнение в `values.yaml` для секретов и ConfigMap из подключённого кластера
- Улучшено автодополнение переменных окружения в Deployment-шаблонах
- Добавлена возможность подключать чарт `kube-prometheus-stack` из палитры действий
